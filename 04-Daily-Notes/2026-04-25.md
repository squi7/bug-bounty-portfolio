# Daily Notes: API Testing Knowledge

Date: 2026-04-25

## Ringkasan Konsep

API (Application Programming Interface) adalah pintu komunikasi antar sistem. Keamanan API sangat krusial karena sering berinteraksi langsung dengan database dan logika backend.

## Tahapan Pengujian (Workflows)

1. Reconnaissance (Pengintaian)
   - Identifikasi endpoint: Mencari lokasi API (contoh: /api/v1/users).
   - Mencari dokumentasi: Mencari file Swagger, OpenAPI, atau direktori seperti /api/docs.
   - Analisis JS: Mencari endpoint tersembunyi di dalam file JavaScript frontend.

2. Interaksi & Manipulasi
   - HTTP Verb Tampering: Mencoba mengganti metode permintaan (GET ke POST, PUT, atau DELETE).
   - Content-Type Testing: Mengubah format data (JSON ke XML) untuk mencari kelemahan parsing.

3. Eksploitasi Logika (Mass Assignment)
   - Mengirimkan parameter yang tidak didokumentasikan (seperti "isAdmin": true) untuk melihat apakah server secara otomatis memperbarui field sensitif di database.

## Temuan Penting / Checklist

- Selalu cek header OPTIONS untuk melihat metode yang diizinkan.
- Perhatikan detail pesan error (verbose errors) yang membocorkan struktur tabel atau path internal.
- Gunakan alat bantu seperti Burp Suite (Repeater, Intruder) dan ekstensi JS Link Finder.

## Strategi Pencegahan

- Terapkan whitelist untuk metode HTTP dan input field.
- Nonaktifkan fitur debug dan amankan dokumentasi API di lingkungan produksi.
- Gunakan pesan error generik untuk meminimalisir kebocoran informasi.
